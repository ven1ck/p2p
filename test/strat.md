# Стратегия внедрения и продвижения приложения в IT TOP


## 1. Позиционирование продукта

### Ценностное предложение
| Для кого | Проблема | Решение |
|----------|----------|---------|
| **Студенты** | Медленная передача файлов через облака, лимиты почты, необходимость регистрации | Мгновенная передача файлов по локальной сети без интернета, регистрации и лимитов |
| **Преподаватели** | Сложность сбора работ у группы, риск потери файлов при отправке через мессенджеры | Централизованный приём файлов с подтверждением, сохранение структуры папок |

### Ключевые преимущества для образовательной среды
```
✅ Работает без интернета — только локальная сеть 9+(В нынешних реалях - полезно)
✅ Не требует аккаунтов, паролей, регистрации
✅ Сохраняет структуру папок (удобно для лабораторных работ)
✅ Подтверждение приёма — гарантия доставки
✅ Открытый исходный код — прозрачность и безопасность
✅ Минимальные требования — работает на старых ПК
```

## 2. Анализ целевой аудитории

### Сегменты пользователей в колледже
```
Студенты 
   • Сценарий: обмен конспектами, проектами, материалами для групповых работ
   • Мотивация: скорость, простота, отсутствие регистрации, VELES

Преподаватели 
   • Сценарий: сбор лабораторных работ, раздача методичек, обмен материалами
   • Мотивация: надёжность, подтверждение получения, организация файлов

Системные администраторы 
   • Сценарий: раздача ПО, обновлений, сбор логов с рабочих мест
   • Мотивация: контроль трафика, безопасность, отсутствие внешних зависимостей
```

## 3. План внедрения (3 фазы)

### Фаза 1: Пилотное внедрение (2–3 недели)
**Цель:** Проверить работоспособность и собрать обратную связь

| Задача | Ответственный | Срок |
|--------|--------------|------|
| Согласование с руководством и IT-отделом | Проект-менеджер | Неделя 1 |
| Установка приложения на 10–15 тестовых ПК (одна группа) | Системный администратор | Неделя 1 |
| Подготовка краткой инструкции (1 страница, скриншоты) | Технический писатель | Неделя 2 |
| Проведение демо-сессии для пилотной группы | Преподаватель-энтузиаст | Неделя 2 |
| Сбор обратной связи через простую форму (Google Forms) | Координатор | Неделя 3 |
| Анализ логов: стабильность, ошибки, производительность | Разработчик | Неделя 3 |

**Критерии успеха фазы 1:**
- [ ] 80% участников пилота смогли отправить/принять файл без помощи
- [ ] Среднее время передачи файла < 10 сек (для 100 МБ в локальной сети)
- [ ] Не более 2 критических ошибок (падения, зависания)

---

### Фаза 2: Масштабирование на поток (4–6 недель)
**Цель:** Охватить 3–5 групп, интегрировать в учебный процесс

**Интеграция в учебный процесс:**
```
📚 Лабораторные работы:
   • Преподаватель раздаёт материалы через приложение → студенты принимают с подтверждением
   • Студенты отправляют выполненные работы → преподаватель видит статус доставки

📁 Групповые проекты:
   • Участники команды обмениваются файлами без загрузки в облако
   • Сохраняется структура папок проекта

🎓 Защита проектов:
   • Быстрая передача презентаций, демо-версий на ПК преподавателя
   • Нет зависимости от скорости интернета в аудитории
```

---

### Фаза 3: Полное внедрение и развитие (2–3 месяца)
**Цель:** Сделать приложение стандартом для локального обмена файлами в колледже

| Направление | Действия |
|-------------|----------|
| **Техническое развитие** | • Добавить журнал отправленных файлов (для преподавателей)<br>• Реализовать «избранные получатели»<br>• Добавить сжатие файлов перед отправкой (опционально) |
| **Организационное развитие** | • Включить приложение в «Пакет первокурсника»<br>• Добавить упоминание в методичках по лабораторным работам<br>• Провести конкурс «Лучший кейс использования» среди студентов |
| **Поддержка и развитие сообщества** | • Создать канал обратной связи в локальном чате<br>• Назначить «амбассадоров» из студентов для помощи новичкам<br>• Публиковать обновления в новостной ленте колледжа |

---



### Сетевая конфигурация
```yaml
# Рекомендуемые настройки для сетевого оборудования колледжа:
udp_discovery:
  port: 50000
  multicast: 255.255.255.255  # Разрешить широковещательный трафик в учебных подсетях

tcp_transfer:
  port: 9999
  direction: inbound/outbound  # Разрешить входящие и исходящие соединения на этом порту

firewall_rules_example:
  - name: "LAN File Transfer UDP"
    protocol: UDP
    port: 50000
    action: ALLOW
    scope: LocalSubnet
    
  - name: "LAN File Transfer TCP"
    protocol: TCP
    port: 9999
    action: ALLOW
    scope: LocalSubnet
```


## 4. Риски и меры по их снижению

| Риск | Вероятность | Влияние | Мера снижения |
|------|-------------|---------|---------------|
| Сопротивление изменениям со стороны преподавателей | Средняя | Высокое | Вовлечение «лидеров мнений», упрощённая инструкция, личная поддержка |
| Блокировка портов сетевым экраном | Низкая | Критическое | Предварительное согласование с IT, документ с правилами для фаервола |
| Конфликт с антивирусом (ложное срабатывание) | Средняя | Среднее | Подписание исполняемого файла сертификатом, добавление в исключения |
| Низкая осведомлённость студентов | Высокая | Среднее | Многоканальная коммуникация, амбассадоры, геймификация (конкурсы) |
| Технические проблемы на старых ПК | Низкая | Низкое | Тестирование на минимальных конфигурациях, режим «экономии ресурсов» |
