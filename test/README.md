# Инструкция по запуску тестирования

## 1. Установка зависимостей
Убедитесь, что виртуальное окружение активировано, и установите `pytest`:
```bash
pip install pytest
```

## 2. Запуск тестов
Откройте терминал в корневой директории проекта и выполните:
```bash
pytest tests/ -v
```
Флаг `-v` включает подробный вывод с именами тестов и статусом выполнения.

## 3. Дополнительные опции запуска
| Команда | Назначение |
|---------|------------|
| `pytest tests/` | Базовый запуск (краткий вывод) |
| `pytest tests/ -v` | Подробный вывод по каждому тесту |
| `pytest tests/ -k "integration"` | Запуск только интеграционных тестов |
| `pytest tests/ -k "core"` | Запуск только unit-тестов |
| `pytest tests/ --tb=short` | Сокращённый вывод трассировки при ошибках |
| `pytest tests/ -x` | Остановка при первом упавшем тесте |

## 4. Интерпретация результатов
- `PASSED` [XX%] : Тест выполнен успешно.
- `FAILED` : Тест провален. В выводе будет указана строка с ошибкой и ожидаемые/полученные значения.
- `ERROR` : Ошибка на этапе подготовки/завершения теста (например, проблемы с фикстурами).
- Итоговая строка `X passed in Y.YYs` подтверждает успешное завершение сессии.

## 5. Автоматизация в CI/CD (опционально)
Для непрерывной интеграции добавьте файл `pytest.ini` в корень проекта:
```ini
[pytest]
testpaths = tests
addopts = -v --tb=short
```
После этого достаточно запускать просто `pytest`.
